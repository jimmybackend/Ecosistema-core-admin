# Checklist final de presentación pública controlada

Este checklist confirma que la presentación puede compartirse públicamente sin exagerar capacidad operativa ni exponer información sensible.

## Estado de módulos
- [ ] Existe matriz de estado por módulo.
- [ ] Billing no se presenta como terminado si está en roadmap.
- [ ] Integrations no se presenta como terminado si está en roadmap.
- [ ] Support no se presenta como terminado si está en roadmap.
- [ ] Jobs/Workers no se presenta como productivo si aún no lo es.
- [ ] Privacy/Compliance está explicado como parcial o roadmap según corresponda.

## Lenguaje comercial
- [ ] El flujo operativo distingue visión y estado actual.
- [ ] No se usan expresiones de “listo/terminado/productivo” para módulos parciales.
- [ ] La narrativa comercial está alineada con la matriz de estado vigente.

## Contacto público
- [ ] Contacto público fue revisado.
- [ ] Teléfono/correo publicados están confirmados como canales oficiales.
- [ ] El canal de contacto en README y `contacto.md` es consistente.

## Privacidad y datos sensibles
- [ ] No hay secretos ni dumps.
- [ ] No hay PII en imágenes o docs.
- [ ] No se publican credenciales, tokens, llaves ni datos reales de clientes.

## Material visual
- [ ] Hay diagramas base.
- [ ] Capturas y diagramas usan datos ficticios o anonimizados.
- [ ] Las imágenes no muestran paneles internos con información sensible.

## Demo
- [ ] Hay guion de demo.
- [ ] La demo separa explícitamente: operativo real vs read-only/dry-run/controlled.
- [ ] No se promete disponibilidad de integraciones no activas por defecto.

## Criterio Go / No-Go

### Go
- documentos alineados,
- estado real claro,
- sin secretos,
- sin promesas falsas.

### No-Go
- módulos parciales presentados como terminados,
- contacto no aprobado,
- capturas con datos reales,
- README desactualizado.
