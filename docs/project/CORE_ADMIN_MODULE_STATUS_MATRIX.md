# CORE Admin Module Status Matrix

Matriz de estado real por módulo del Core Admin, basada en rutas declaradas, servicios/repositorios existentes y flags de `.env.example`.

> Criterio de este documento: si existe documentación previa pero no hay rutas/servicios visibles en el código de este repositorio, el estado se marca como **documental / roadmap**.

## Leyenda de estado
- **Implementado estable**: módulo con rutas funcionales y sin etiqueta dry-run/read-only predominante.
- **Read-only**: consultas/listados/detalle sin escrituras funcionales.
- **Dry-run**: simulación sin escritura final o sin ejecución real externa.
- **Controlled por flags**: la ejecución real depende de flags `*_ENABLED`, `*_WRITE`, etc.
- **Documental / roadmap**: existe narrativa/documentación, pero no implementación operativa confirmada en rutas.
- **Pendiente / no implementado**: no se confirma implementación en este PR.
