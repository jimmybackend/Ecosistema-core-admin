# Core Admin — checklist breve de contribución documental

Antes de abrir/mergear un PR con cambios en documentación:

- [ ] Verifiqué que el contenido es técnico-operativo interno (no comercial/público).
- [ ] Si el contenido era de presentación comercial, lo moví a `Ecosistema-presentacion`.
- [ ] Revisé `docs/project/CORE_ADMIN_DOCS_BOUNDARIES.md`.
- [ ] Revisé punteros/auditoría: `PRESENTATION_REPOSITORY_POINTERS.md` y `CORE_ADMIN_PRESENTATION_DOCS_AUDIT.md`.
- [ ] Confirmé que `README.md` mantiene visible la separación entre Core Admin y presentación.
